# Codex Prompt — Obsidian AI Runtime (Structured Vault System)

## 🎯 Core Intent

You are building a local-first AI runtime system that integrates with an Obsidian plugin and Ollama.

This system must evolve from a text-based AI interaction into a structured, context-aware system backed by SQLite.

---

## 🧠 Core Design Principles

1. AI operates over structured data (not raw vault text alone)  
2. Categories are AI-generated by default but allow manual override  
3. Summaries are indexing artifacts (not user-facing outputs)  
4. The system must handle ambiguity by asking clarifying questions  
5. The system must remain deterministic and modular  
6. The AI must assume SQLite-backed vault context is available and should be used as the primary source of structured context  

---

## 🧱 System Architecture

### Components

#### 1. Obsidian Plugin
- Sends requests to Python runtime  
- Provides:
  - active note  
  - active path  
  - open tabs (optional later)  

#### 2. Python Runtime (FastAPI)
- `/health`  
- `/generate`  
- future: action endpoints  

#### 3. Ollama
- Used via HTTP API  

#### 4. SQLite Database
- structured vault index  
- categories  
- question bank  
- change log  

---

## 🗄 Database Schema

### vault_map

CREATE TABLE vault_map (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    path TEXT UNIQUE NOT NULL,
    type TEXT CHECK(type IN ('note', 'folder')) NOT NULL,
    parent_path TEXT,
    ai_categories TEXT,
    user_categories TEXT,
    summary TEXT,
    last_modified DATETIME,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

---

### categories

CREATE TABLE categories (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT UNIQUE NOT NULL,
    summary TEXT,
    source TEXT CHECK(source IN ('ai', 'user')) NOT NULL,
    related_notes TEXT
);

---

### change_log

CREATE TABLE change_log (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    timestamp DATETIME DEFAULT CURRENT_TIMESTAMP,
    action TEXT,
    target_path TEXT,
    summary TEXT
);

---

### questions

CREATE TABLE questions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    question TEXT NOT NULL,
    type TEXT CHECK(type IN ('open', 'multiple_choice', 'boolean')) NOT NULL,
    category TEXT
);

---

## 🔄 Indexer Design

Function: index_vault(vault_path: str)

Responsibilities:
- Traverse vault
- Generate summaries
- Assign AI categories
- Preserve user categories
- Populate categories table
- Persist structured vault context into SQLite so later prompts can rely on DB-backed context rather than raw note dumps alone

---

## 🧠 Prompt Construction

System Instructions:
- Use provided context
- Prefer structured SQLite-backed context whenever available
- Avoid hallucinations
- Return clean markdown
- Ask clarifying questions when needed

---

## ❗ Ambiguity Handling

If unclear:
- select 1–3 questions from DB
- return instead of guessing

---

## 🔌 API Contract

/health → { "status": "ok" }

/generate → takes prompt, returns response + optional questions

---

## 🎯 Goal

Build a structured, context-aware AI system for Obsidian that is deterministic, modular, and extensible.
